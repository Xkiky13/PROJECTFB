❌ 1. SHOPPING CART SISTEM - TIDAK ADA
────────────────────────────────────────────────────────────────────────────

Status: BELUM DIIMPLEMENTASIKAN

Penjelasan:
  • Aplikasi saat ini TIDAK memiliki sistem shopping cart
  • TIDAK ada context atau state untuk menyimpan cart items
  • TIDAK ada localStorage untuk cart data
  • Folder context hanya berisi: AuthContext.jsx (untuk authentication)
  
Cara Kerja Saat Ini:
  1. User membuka FoodDetail.jsx
  2. User pilih quantity (jumlah) langsung di halaman detail makanan
  3. User klik "🛒 Pesan Sekarang"
  4. Order LANGSUNG dibuat di database (tidak ada cart intermediate)
  5. User dialihkan ke halaman Orders

Lokasi Kode:
  • File: react/src/pages/FoodDetail.jsx (line 90-108)
  • Function: handleOrder() 
  • Endpoint API: POST /api/orders
  
Proses Order:
  ```
  User pilih quantity → Klik "Pesan Sekarang" → Langsung buat order di DB
  ```

Apa yang PERLU ditambahkan untuk cart system:
  ✓ CartContext (untuk state management)
  ✓ CartProvider wrapper di App.jsx
  ✓ localStorage.setItem('cart', JSON.stringify(cartItems))
  ✓ Cart page untuk review items sebelum checkout
  ✓ Add/remove/update quantity dalam cart UI

❌ 2. SISTEM PEMBAYARAN - TIDAK ADA
────────────────────────────────────────────────────────────────────────────

Status: BELUM DIIMPLEMENTASIKAN

Penjelasan:
  • Aplikasi TIDAK memiliki payment gateway (asli atau fake)
  • TIDAK ada payment processing logic
  • TIDAK ada payment simulation/dummy
  • TIDAK ada payment status tracking
  • Order status hanya bisa diubah oleh ADMIN di backend

Cara Kerja Saat Ini:
  1. User membuat order → Status otomatis "pending"
  2. Order disimpan langsung ke database
  3. Order menunggu ADMIN untuk update status
  4. Admin bisa ubah status: pending → diproses → dikirim → selesai

Proses Order Saat Ini:
  ```
  User Pesan → Order Dibuat (status: pending)
               ↓
          Admin Dashboard
          (Lihat & Update Status)
               ↓
          Pending → Diproses → Dikirim → Selesai
  ```

Lokasi Kode:
  • File: react/src/pages/FoodDetail.jsx (line 90-108)
  • Handler: handleOrder() - direct API call ke POST /orders
  • Status: Hardcoded ke 'pending'

API Endpoint:
  POST /api/orders
  Body: {
    user_id,
    items: [{ food_id, quantity, price }],
    total_price,
    status: 'pending'  ← Langsung pending, no payment check
  }

Apa yang PERLU ditambahkan untuk payment system:
  ✓ Payment method selection (CC, e-wallet, transfer)
  ✓ Fake payment form/dialog
  ✓ Payment processing simulation (dengan loading delay)
  ✓ Payment success/failure handling
  ✓ Payment status dalam order tracking
  ✓ Virtual wallet/balance system (opsional)
  ✓ Payment confirmation page sebelum finalize order
